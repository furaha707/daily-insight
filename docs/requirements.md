# 📑 [기획 명세서] 매일 아티클 하나씩 추천해주는 데일리 인사이트 서비스

## 1. 프로젝트 개요 (Overview)

- **프로젝트명**: dailyInsight
- **배경 및 문제 정의**:
    - IT 아티클을 매일 읽으면 기획 스킬 향상에 도움이 된다. 그러나 매일 고르는 것도 일이고, 까먹을때도 허다하다.
    - 사용자는 관심 카테고리를 선택한 후 버튼을 누르면 클로드 코드가 카테고리에 해당되지만 중복되지 않은 아티클을 선정해서 출력해준다.
- **목표**: 매일 하루에 한 번씩 이 서비스를 사용할 수 있게 해서 읽지 못한 아티클이 없게 함으로 학습 습관을 형성한다.

---

## 2. 핵심 기능 요구사항 (Requirements)

- **사용자 플로우**: 메인 화면에서 카테고리를 체크박스로 선택함 -> 아티클 신청하기 버튼 선택 -> 모달로 신청 완료가 노출되면서 카테고리에 해당하는 제목과 링크가 노출됨
- **백엔드(Supabase) 연동**:
    - **데이터 구조**: DB는 총 users, articles, sent_log 로 구성한다. users는 id, 선택 카테고리 그리고 articles는 아이디, 제목, 링크, 카테고리 그리고 sent_log는 user 아이디, article 아이디, 발송시간, 발송여부 이 외에 더 필요한 데이터 컬럼이 있다면 추가한다.

---

## 3. UI/UX 와이어프레임 및 디자인 가이드

### 3-1. 메인 화면 (Home)

- **타이틀**: "새로운 인사이트를 얻을 수 있는 아티클을 선정해드려요"

- **카테고리 선택 영역**: "커뮤니케이션, 문제해결역량, Zapier, Jira, 비즈니스, 마케팅, 디자인, 기획, AI, Figma, 데이터, 개발프로세스 협업, 취업, AI툴, UI/UX" (다수 선택 가능)
- **CTA 버튼**: `[신청하기]` (애니메이션 효과 포함)
